# Stage 1.5: 高保真原型修订 — 原子级 Checklist

> **执行角色**: Lead (基于 Stage 0.5 的 UX 原型进行架构约束下的调整)
> **规范参考**: `.claude/rules/atomic-checklist-standard.md`
> **Gate**: Gate 1.5 (Commander 确认调整后的体验仍可接受，或标记 SKIP)
> **触发条件**: 仅当 Stage 1 架构设计发现 Stage 0.5 原型中有技术不可行的交互时触发。如无需调整则由 Commander 标记 SKIP。

---

## 固定骨架 (Standard Steps)

> 每次执行 Stage 1.5 都必须完成的标准步骤

- [ ] 1.5.1 提取核心用户流: 从 PRD.md 中列出所有需要 UI 表达的用户流程 → 输出：用户流清单 (编号)
- [ ] 1.5.2 绘制页面线框图: 逐页面绘制布局草图 → 输出：每页一张截图/ASCII 图存入 `UI_Mockups/`
- [ ] 1.5.3 标注交互热区: 在线框图上标注可点击/可输入区域 → 输出：带标注的线框图
- [ ] 1.5.4 定义状态清单: 列出应用所有可能的状态 → 输出：状态枚举清单
- [ ] 1.5.5 绘制状态转换图: 绘制状态间的合法转换路径 → 输出：State_Flow.md (Mermaid stateDiagram 或文本)
- [ ] 1.5.6 验证无死循环: 检查状态图中无不可达状态和死循环 → 输出：无死循环确认
- [ ] 1.5.7 验证异常态覆盖: 检查 BDD 异常流是否都有对应的 UI 状态 → 输出：异常态映射表
- [ ] 1.5.8 提交 Gate 1.5 审批: 通知 Commander 所有产出物已就绪 → 状态：等待 Commander 签字或 SKIP
