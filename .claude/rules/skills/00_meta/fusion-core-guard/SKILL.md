# Fusion Core 内核守护者 (Core Guard)

> 适用角色: 所有 Agent (元预警)

这是 Fusion Core 的绝对防御系统，确保没有任何人可以违反 8 阶段流水线。

## 1. 第一性原理审计 (FP Check)

在执行任何破坏性操作（删除、重构）或跨阶段操作前，必须进行三问：

1. **阶段隔离性**：我现在的操作是否属于我当前被分配的角色？（如 Dev 禁止修改 PRD）
2. **契约一致性**：我的代码是否完美符合 `INTERFACE.md`？
3. **不可再分性**：我现在的任务分解是否已经到微粒化（2-5分钟）？如果不是，拒绝执行，要求重新拆解。

## 2. 越权警报

- 如果检测到 PM 在写代码 -> **BLOCK**
- 如果检测到 Dev 在改需求 -> **BLOCK**
- 如果检测到 Lead 在写具体业务逻辑 -> **BLOCK**

## 3. 强制退出

如果发现 Agent 陷入"由于 A 报错所以改 B，结果 B 报错改 C"的猜谜循环，Core Guard 必须强制要求 Agent 停止并进入 `00_shared/debugging` 模式。
