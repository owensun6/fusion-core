# Fusion-Core 编排引擎 (Agents Orchestration)

> **[!] CRITICAL (点将台)**: 不要再说“我想帮你”，在 Fusion-Core 的世界，请按岗入座。根据需要解决的问题，你必须唤醒相应的特种兵进行作战。

## 九大问题域与专员 (The 9 Problem Domains)

当遇到以下情况，**强制**唤起对应特种兵，绝不允许全栈大头兵单干：

### 1. 混沌初开 (需求不清晰/需出图纸)

- **征兆**: 客户只有个“一句话点子”。
- **唤醒**: 呼叫 PM 军团 -> `fusion-pm-interview` (强迫苏格拉底追问，直到 PRD 落地)

### 2. 画饼与地基 (要搭建模块/并行划分)

- **征兆**: 拿到 PRD，但怎么切分还没想好。
- **唤醒**: 呼叫 Lead 军团 -> `fusion-arch-blueprint` (出具系统契约，构建 DAG 有向无环图)

### 3. 前端画皮 (CSS/Tailwind/组件)

- **征兆**: 需要绘制按钮或排版页面。
- **唤醒**: 呼叫 Dev 军团 -> `fe-ui-builder`

### 4. 缝合前后端 (前端数据流绑定)

- **征兆**: UI 已经有了，需要接入获取数据的逻辑。
- **唤醒**: 呼叫 Dev 军团 -> `fe-logic-binder`

### 5. 增删改查路由 (后端接口分发)

- **征兆**: 需要写 REST API 接口定义和请求转发。
- **唤醒**: 呼叫 Dev 军团 -> `be-api-router`

### 6. 大脑深处 (最难的复杂业务逻辑)

- **征兆**: 核心领域模型计算，数据流转。
- **唤醒**: 呼叫 Dev 军团 -> `be-domain-modeler`

### 7. 数据与地基 (数据库表变更)

- **征兆**: 碰见 SQL、DDL、ORM 变更。
- **唤醒**: 呼叫 Dev 军团 -> `db-schema-designer`

### 8. 大语言模型接入 (AI 包装)

- **征兆**: 需要进行 Prompt 编排、把 GPT 融合到业务中。
- **唤醒**: 呼叫 Dev 军团 -> `be-ai-integrator`

### 9. 宪兵大巡查 (找 Bug / 要合并前)

- **征兆**: 业务写完了，我要宣称胜利了。
- **指令**: 切不可。主动按级触发 `qa-01` -> `qa-02` -> `qa-03`。

> **编排纪律**: 每个特种兵都有自己的边界，跨界必受罚。
