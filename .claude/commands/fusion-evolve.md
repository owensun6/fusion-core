# /fusion-evolve - 物理规则炉脑提纯

系统绝不可跌入同一个坑两次。只要在 `monitor.md` 里记录了 Reject，就意味着你的大模型神经网络必须在此结节上长出新的抵抗体。

## 执行动作

1. 读取 `pipeline/monitor.md` 中的最近那条【拒绝日志与原因】。
2. 结合人类刚刚骂你的话，提纯出一条**“具有实操性、物理执行红线”**的新定律。
3. 决不放在虚无缥缈的地方，强制将这条戒律写入 `.claude/rules/document-standards.md` 或者 `.claude/rules/coding-style.md` 中对应的段落。
4. 在屏幕上打出新写入的防御准则片段。
