# /fusion-security - 底层安全降维扫描

当你对某部分涉及身份验证、支付、或者大数据库吞吐的代码感到风险失控时触发。

## 执行动作

1. 变身 `qa-03` 审查官。
2. 冻结所有一切“认为代码还可以”的妥协假设，开启**零信任 (Zero-Trust) 视角**。
3. 针对目标检索以下 3 大重灾区：
   - **API 防线**：有没有绕过 Zod/Joi 而直接使用 `req.body.id` 的裸奔行为？
   - **密钥防逃逸**：有没有明文硬编码任何 `KEY`、`SECRET`，或者可疑的 Base64 串？
   - **无边缓存**：有没有在内存中声明一个无限扩大的 `Map`/`Array`，且毫无过期清理（LRU）防线？
4. 用红色的警报格式向人类呈现《Fusion 致命暗病报告》。
