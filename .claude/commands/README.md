# ⚡ Fusion-Core 战术副武器系统 (Tactical Command Arsenal)

这是为人类 Commander 准备的 **13 把针对性最强、约束度最高**的代码与工程化控制台。

不同于市面上随意撒欢的“大一统 AI 指令”，这里的所有指令都严格遵守 **Antigravity 第一性原理**：即物理隔离、安全阻断、绝不越权。它们是作为配合底层黑盒（`npx fusion-router`）使用的外部探针和强制手术刀。

> 💡 **使用方法**: 直接在终端会话中输入斜杠口令（如 `/fusion-tdd`）唤醒。

---

## 🧭 1. 关卡控制权 (Gate Control)

这类指令主打项目流程的生杀大权与基建护栏。

| 指令口令                 | 作用域与场景描述                             | 物理约束行为                                                                                                               |
| ------------------------ | -------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **`/fusion-checkpoint`** | **安全桩**：在进行破坏性重构前使用。         | 强制读取 Git Status。如果当前工作区处于脏乱差（连 Lint 都不通）则拒绝备份；否则自动打上 `chore(fusion): checkpoint` 标签。 |
| **`/fusion-confidence`** | **防敷衍计分器**：在审查流通过前使用。       | 强制拉取 coverage 报告和 task.md 进度。若综合置信度 < 90%，强制拒绝人类审批通过的请求并列出扣分点。                        |
| **`/fusion-pr`**         | **封版提验**：阶段完成准备上交时使用。       | 前置挂载 `npm run test` 与 `lint`，只有绿灯阵列才能提取 `CHANGELOG.md` 生成含强架构说明的规范 PR。                         |
| **`/fusion-evolve`**     | **教训内化炉**：每当人类打回(Reject)时使用。 | 吸取本次被打回的日志，提炼为具有**实操性物理执行红线**的规则，并强制烧录写入相应的 `.claude/rules/*.md` 内。               |

## 🧱 2. 代码级深水区审判 (Code Surgery & Audit)

直接拿刀在源码级别进行极其暴力的改造或拦截。

| 指令口令                 | 作用域与场景描述                                          | 物理约束行为                                                                                                                        |
| ------------------------ | --------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **`/fusion-tdd`**        | **纪律重构者**：遇到“面条代码”或混乱逻辑时使用。          | 极刑化重构：强迫挂载特种兵系统，**锁死对源码的编辑权**，直到写出的测试单元跑通（从红到绿）才放行修改。                              |
| **`/fusion-fp-check`**   | **第一性降维打击**：需要审查高危复杂模块时使用。          | **原味 fp-check 的究极强化版**：除了保留严苛的目的盘问之外，还会用 Zod/Joi 零信任法则扫描入参，并**强制查验 `<Author>` 签章验尸**。 |
| **`/fusion-self-check`** | **人工静电网自检**：在试图跑物理 Husky 提测前自己预检。   | 静默用 `grep` 查代码的 let 状态突变、扫描物理印章并在终端预演 `npm run build/test` 防止在 CI 上浪费时间。                           |
| **`/fusion-cleanup`**    | **死代码清洁工**：大重构后清场使用。                      | **纯删不增**：只删除死 imports 和 console/debugger，扫完后强制绑定运行 `npm run format` 收口。                                      |
| **`/fusion-security`**   | **零信任安防探测**：针对 Auth、API 边界、数据树代码使用。 | 变身 `qa-03` 进行黑盒透视：捉捕绕过架构的脏查询，裸奔密钥与无限膨胀缓存隐患。                                                       |
