## 执行序列

### Step 1: 汇总所有屏幕（来自 Feature_Screen_Map）

列出屏幕目录，确认每个屏幕的 notes.md 都已存在。

### Step 2: 逐屏写出 UI_CONTRACT 章节

每个屏幕输出一个标准章节，格式如下：

```markdown
## 屏幕: [屏幕名称]

**对应 F-ID**: F1.1, F1.2
**用户角色可见性**: 普通用户（未登录时不可见）
**入口路径**: 登录页 → 点击「注册」→ 本屏幕

### 布局结构

| 区域 | 区域名称 | 内容描述                        | 交互规则           |
| ---- | -------- | ------------------------------- | ------------------ |
| 顶部 | 页头     | Logo + 返回链接                 | 点击 Logo → 回首页 |
| 主体 | 注册表单 | 字段：用户名/邮箱/密码/确认密码 | 提交前校验所有字段 |
| 底部 | 辅助链接 | "已有账号？去登录"              | 点击 → 跳转登录页  |

### 组件清单

| 组件ID   | 组件名称 | 描述         | 状态变体              |
| -------- | -------- | ------------ | --------------------- |
| C-REG-01 | 输入框   | 用户名输入   | 默认/聚焦/错误/成功   |
| C-REG-02 | 提交按钮 | 完成注册     | 默认/加载中/禁用      |
| C-REG-03 | 错误提示 | 字段校验失败 | 显示（红色文字）/隐藏 |

### 状态规则

| 状态       | 触发条件         | 用户看到什么                               |
| ---------- | ---------------- | ------------------------------------------ |
| 初始状态   | 页面加载完成     | 空白表单，提交按钮灰色不可点               |
| 填写中     | 用户输入任意字段 | 提交按钮变为可点击蓝色                     |
| 提交中     | 点击提交         | 按钮变为「注册中…」+旋转图标，表单不可编辑 |
| 注册成功   | 服务端确认       | 跳转到「注册成功」页                       |
| 注册失败   | 字段格式错误     | 对应字段下方显示红色错误说明               |
| 邮箱已注册 | 服务端返回冲突   | 邮箱字段下方显示「此邮箱已被注册」         |

### 导航规则

| 操作           | 跳转目标            | 条件   |
| -------------- | ------------------- | ------ |
| 注册成功       | 注册成功页 / 主界面 | 无条件 |
| 点击返回       | 上一页              | 无条件 |
| 点击「去登录」 | 登录页              | 无条件 |
```

### Step 3: 写 Design Token 规范（视觉方向约束）

```markdown
## Design Token

> fe-ui-builder 必须严格遵循以下视觉规范。这不是建议，是约束。

### 配色方案

| Token 名称 | 色值 | 用途 |
|-----------|------|------|
| --color-primary | #[待定] | 主操作按钮、链接、焦点状态 |
| --color-primary-light | #[待定] | 悬停态、浅色背景 |
| --color-surface | #[待定] | 卡片/面板背景（必须为白色或浅色） |
| --color-background | #[待定] | 页面整体背景（必须为白色或接近白色，禁止暗色） |
| --color-text-primary | #[待定] | 正文 |
| --color-text-secondary | #[待定] | 辅助说明文字 |
| --color-error | #[待定] | 错误提示 |
| --color-success | #[待定] | 成功状态 |

### 字体体系

| 层级 | 字体 | 字号 | 字重 |
|------|------|------|------|
| 标题 H1 | [待定] | [待定] | Bold |
| 标题 H2 | [待定] | [待定] | SemiBold |
| 正文 Body | [待定] | [待定] | Regular |
| 辅助 Caption | [待定] | [待定] | Regular |

### 间距与圆角

| Token | 值 | 用途 |
|-------|----|------|
| --spacing-xs | [待定] | 组件内部间距 |
| --spacing-sm | [待定] | 相关元素间距 |
| --spacing-md | [待定] | 区域间距 |
| --spacing-lg | [待定] | 页面级间距 |
| --radius-sm | [待定] | 按钮/输入框 |
| --radius-md | [待定] | 卡片 |
| --radius-lg | [待定] | 弹窗/面板 |

### 视觉质感方向

- **整体风格**: [一句话描述，如"Clinical Elegance — 权威而不冰冷，精致而不花哨"]
- **色调基底**: 白色/浅色背景（医疗行业客户，禁止 dark theme）
- **阴影层次**: [无阴影 / 微妙阴影 / 明显层次]
- **背景质感**: [纯色 / 渐变 / 纹理]（底色必须为白或接近白色）
- **动画基调**: [无动画 / 微妙过渡 / 活泼弹性]
```

**[待定] 由 UX Designer 根据项目特性填写具体值。** Commander 未提供品牌规范时，UX Designer 基于产品定位自主决定，在 Gate 0.5 时由 Commander 确认。

### Step 4: 写全局契约规则（跨屏幕通用约定）

```markdown
## 全局契约

### 加载状态标准

- 所有异步操作（数据加载/提交）期间：操作按钮禁用 + 显示旋转图标
- 加载超过 3 秒：显示提示文字（具体文案见各屏幕描述）

### 空状态标准

- 列表为空时：显示图标 + 说明文字 + （可选）引导操作按钮
- 不允许显示空白区域（空白 = 用户不知道发生了什么）

### 错误提示标准

- 字段级错误：显示在字段正下方，红色文字
- 全局错误（操作失败）：顶部 Toast 提示，3 秒后自动消失
- 错误文字必须是用户能理解的自然语言，不出现错误码

### 文字规范

- 所有按钮文字：动词开头（"登录" / "保存" / "删除"）
- 所有提示文字：完整句子，不用技术术语
- 占位符文字（Placeholder）：说明预期格式（如"example@email.com"）
```
