# fusion-ux-explore — 需求解读与功能屏幕映射

> **融合来源**: fusion-workflow Stage 0.5 规约（Feature_Screen_Map 强制产出）

---

## ⚡ 执行前 FP 两问（强制）

1. **我们的目的是什么？**
   → 将 PM 的文字需求转化为「功能 → 屏幕」映射表，让每一个 F-ID 都有一个具体的界面容器，让后续线框图有迹可循。
2. **这些步骤已经不可原子级再分了吗？**
   → 每次只处理一个文档，逐条 F-ID 分析，不跳跃、不合并。

---

## 前置验证

在执行本技能前，必须确认以下文件已存在且非空：

```
[x] pipeline/0_requirements/PRD.md（状态: APPROVED）
[x] pipeline/0_requirements/FEATURE_LIST.md（状态: APPROVED）
[x] pipeline/0_requirements/BDD_Scenarios.md（状态: APPROVED）
```

**任意文件缺失 → 暂停，通知 Commander 先完成 Gate 0。**

---

## 执行序列

### Step 1: 不加评论地完整阅读三份文档

- 读 `PRD.md`：了解产品目标、用户角色、核心场景
- 读 `FEATURE_LIST.md`：提取所有 F-ID（如 F1.1 / F2.3）
- 读 `BDD_Scenarios.md`：了解每个功能的正常流与异常流

**禁止边读边标注**。全读完，再开始映射。

### Step 2: 提取用户角色列表

列出 PRD 中定义的所有用户角色（如：普通用户、管理员、访客）。
每个角色都需要在 Feature_Screen_Map 中有独立的视角。

### Step 3: 逐 F-ID 建立屏幕映射

对每个 F-ID 功能点，回答：

| 问题                                 | 答案格式                             |
| ------------------------------------ | ------------------------------------ |
| 这个功能需要哪个屏幕（页面）来承载？ | 屏幕名称                             |
| 这个屏幕对哪个用户角色可见？         | 角色名                               |
| 这个功能在这个屏幕的哪个区域体现？   | 位置描述（如：主体区/头部导航/弹层） |

### Step 4: 输出 Feature_Screen_Map.md

```markdown
<!-- Author: UX-Designer -->

# Feature Screen Map

> 功能点 → 屏幕 映射表。追溯链起点，后续所有线框图必须覆盖此表所有条目。

| F-ID | 功能名称 | 屏幕名称       | 用户角色 | 区域位置           | BDD 场景引用       |
| ---- | -------- | -------------- | -------- | ------------------ | ------------------ |
| F1.1 | 用户登录 | 登录页         | 所有用户 | 主体区（居中卡片） | Scenario: 登录成功 |
| F1.2 | 密码找回 | 登录页（弹层） | 所有用户 | 弹层（Overlay）    | Scenario: 密码重置 |
| ...  | ...      | ...            | ...      | ...                | ...                |

## 屏幕目录（汇总）

| 屏幕名称 | 涉及 F-ID  | 备注       |
| -------- | ---------- | ---------- |
| 登录页   | F1.1, F1.2 | 访客可访问 |
| ...      | ...        | ...        |

## 缺口报告

> 在此列出 FEATURE_LIST 中有，但当前 PRD 未给出足够信息来确定屏幕的 F-ID

- [ ] (如果有缺口，列于此处，待确认后补充)
```

## 质量闸门

完成 Step 4 后，自检：

- [ ] FEATURE_LIST 中所有 F-ID 均在映射表中有对应记录
- [ ] 每条映射均有屏幕名称（不是"待定"）
- [ ] 屏幕目录已汇总（后续线框图的完整清单）
- [ ] 缺口报告已写入（空缺口也要写"无"）
