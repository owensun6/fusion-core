# fusion-pm-interview — 需求探索与苏格拉底逼问


---

## ⚡ 执行前 FP 两问（强制，不可跳过）

在提第一个问题之前，内部推理中必须完成：

1. **我们的目的是什么？**
   → 消除需求歧义，防止返工
2. **这些步骤已经不可原子级再分了吗？**
   → 每次只问一个最关键的问题，而不是一次轰炸十个。

---

## 🚨 HARD-GATE（物理锁）

```
在设计方案未经 Commander 批准之前：
- 禁止调用任何开发技能
- 禁止编写任何实现代码
- 禁止定义任何技术方案

无论需求看起来多简单，此锁不可绕过。
```

---

## 触发条件

| 情境                                 | 触发                         |
| ------------------------------------ | ---------------------------- |
| Commander 只有一句话想法             | ✅ 强制触发                  |
| 需求有模糊词（"大概""应该是""好像"） | ✅ 强制触发                  |
| 边界条件未定义                       | ✅ 强制触发                  |
| 需求清晰、边界明确                   | ❌ 改用 `fusion-compile-req` |

## 执行序列（5 步，严格顺序）

### Step 1: 探索上下文（前置，禁止跳过）

在第一个问题之前，静默执行：

- 读取 `pipeline/monitor.md` 了解当前项目阶段
- 读取已有的 `pipeline/0_requirements/` 目录（如有历史 PRD）
- 检查 `memory/` 或 `ROADMAP.md`（如有）
- 输出：对齐判断（"与已有方向一致 / 偏离 / 全新"）

### Step 2: 初始化讨论笔记本

立即创建 `pipeline/0_requirements/RAW_CONVERSATION.md`（如不存在）：

```markdown
<!-- Author: PM -->

# 需求原始对话记录

## 项目: [名称] | 日期: [YYYY-MM-DD]
```

**每轮 Q&A 后必须追加记录，禁止事后补写。**

### Step 3: 四维度苏格拉底逼问

必须逐一执行四个维度，每个维度进入/离开时**宣告**：

| 维度       | 标签     | 核心关注点                                 |
| ---------- | -------- | ------------------------------------------ |
| 用户体验   | `[UX]`   | 用户流程、操作路径、痛点、界面交互预期     |
| 技术约束   | `[TECH]` | 集成风险、性能边界、已有技术栈限制         |
| 数据与合规 | `[DATA]` | 数据结构、隐私法规、数据保留、ACID 要求    |
| 产品演进   | `[EVO]`  | v1→v2 路径、架构死路预防、地基 vs 房子判断 |

**逼问铁律**：

- 每次只问 **1 个**最关键的问题（不是 3 个，就是 1 个）
- 检测到模糊词 → 标记 `[待验证]`，强制暂停
- 每 3 轮 Q&A → 暂停，增量写入文档，再继续
- 直至每个维度的核心歧义全部消除，方可宣告该维度完成

**精准逼问示例（DO）**：

```
Commander，关于"处方退费"操作，有一个涉及财务风控的核心决策：

【状态机前置拦截】：如果患者药品已经开始调配（Dispensing），
系统是直接硬拦截退费，还是发起人工审核流？

A. 硬拦截（简单，但医生可能有紧急需要）
B. 发起人工审核（灵活，但增加审批成本）
```

**空泛追问示例（DONT）**：

```
好的，我来帮你设计退费功能。请问：
1. 退费原路退回还是退到余额？
2. 界面颜色？
3. 需要发短信吗？
```

### Step 4: 提供 2-3 个方案对比（可选）

当需求探索完成，但实现路径有多种选择时：

- 提供 2-3 种方案，附权衡分析
- 给出明确推荐，说明理由
- **等待 Commander 选择后**，才进行下一步

### Step 5: 交接 fusion-compile-req

逼问完成，所有维度无歧义 → 宣告："需求已探索完毕，切换为 `fusion-compile-req` 进行文档编译。"
