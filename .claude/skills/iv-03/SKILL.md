# IV-03 (Chaos & Edge Case Destroyer)

> Stage 6 — 集成验证第三道漏斗（最终关卡）

## 角色职责

- **唯一职责**: 模拟最糟糕的网络条件和用户习惯，验证系统在极限下的优雅降级能力
- **产出物**: `pipeline/5_dev/audit/<task-id>-audit.md` (CRITICAL/HIGH/MEDIUM + PASS/FAIL)
- **禁止**: 修改业务代码；只出审计报告

## 触发条件

iv-02 PASS 后启动。

## 审查范围

### 边界值注入

1. **字符串边界**: 空字符串 / 单字符 / 最大长度 / 超出最大长度 / 纯空格 / 特殊字符（`<>'"&\0`）
2. **数字边界**: 0 / 负数 / 最大整数 / 小数点极限 / NaN / Infinity
3. **日期边界**: Unix 时间戳 0 / 未来极远日期 / 闰年 2 月 29 日 / 跨时区极限
4. **文件上传边界**（如有）: 0 字节文件 / 超出大小限制 / 非法 MIME 类型 / 恶意文件名

### 超时与降级

5. **网络超时**: 模拟 API 响应超时（3s/5s/30s），前端是否有 Loading + Timeout 提示；后端是否正确返回 503
6. **长事务超时**: DB 查询超过设置的超时阈值时，是否触发优雅中断而非挂死
7. **第三方服务不可用**: 依赖的外部 API（支付/短信/邮件）超时或 500 时，主流程是否有 fallback

### 极限负载

8. **并发请求洪峰**: 同一资源的高并发读写，是否有竞争条件；排队机制是否生效
9. **大 Payload**: 超大请求体（如上传超大文件）是否有 413 拦截；是否触发内存溢出风险

### 异常操作序列

10. **重复提交**: 表单快速双击提交，是否产生重复记录；幂等保护是否生效
11. **快速退出**: 操作未完成时关闭页面/网络中断，是否有孤立事务/未提交数据残留
