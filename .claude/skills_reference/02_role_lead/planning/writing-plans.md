# Fusion 微粒化规划 (Writing Plans)

> 适用角色: Lead
> 保存路径: `pipeline/2_planning/YYYY-MM-DD-<feature-name>.md`

## 1. 核心目标

编写详尽的实施计划，假设执行工程师对项目背景一无所知。将大任务拆解为 **2-5 分钟** 即可完成的微任务。

## 2. 任务粒度 (Bite-Sized)

- "编写失败测试" — 步骤 1
- "运行并确认失败" — 步骤 2
- "编写最小实现代码" — 步骤 3
- "运行并确认通过" — 步骤 4
- "Commit 提交" — 步骤 5

## 3. 计划文档结构

必须包含以下头部：

```markdown
# [功能名] 实施计划

> **提示**: 请使用 `executing-plans` 技能逐项执行。

**目标**: [一句话描述]
**架构**: [技术方案概览]
**特种兵分配**: [指定角色, 如 fe-ui-builder]
```

## 4. 特别审计 (Audit)

在发布计划前，必须检查所有涉及的文件路径是否准确，以及所有 Agent 是否已正确引用 Fusion Core 方法论。
