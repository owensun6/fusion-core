# 02_Intent_Extract (意图萃取与 BDD 契约固化)

> **目标**: 将苏格拉底逼问后得到的内容，转化为机器可读的 BDD 契约和 PRD。

## 触发条件与协作角色

- **调用时机**: 当 Commander 对苏格拉底式逼问完成了所有决断，并且无更多漏洞需要确认时。
- **上游依赖**: `01_Socratic_Ask.md` 收集到的用户解答。
- **下游交接**: 触发 Gate 0 让 Commander 签字。下游受体为 Lead 军团。

## 产出物标准规范 (Deliverable Specs)

你必须且只被允许产出以下两个文件的骨架结构：

1. `PRD.md`: 必须包含背景、用例清单、非功能性需求（如并发量预估、等保安全）。
2. `BDD_Scenarios.md`: 必须严格使用 `Given-When-Then` Gherkin 语法。

---

## 示例对比 (DO / DON'T)

### 场景：编写开药扣减库存的场景

#### ❌ DON'T - 模糊的自然语言描述

```markdown
# 扣减库存场景

当医生开处方时，如果系统还有库存，就扣除对应的量。如果库存不够，就报错提示医生，不让他开这药。
```

**问题**: 无法被测试框架自动化执行，没有数据边界，也没有明确的异常 Code 约定。

#### ✅ DO - 严格的 Gherkin 契约约束

```gherkin
Feature: 处方单库存强校验阻断机制

  Scenario: [Success] 库存充足情况下的正常扣减与锁表
    Given 药房主库中药品 "A司匹林" 当前总可用库存为 100 盒
    And 当前医生 "Doc_001" 拥有处方权
    When 医生发起处方写入，要求包含 2 盒 "A司匹林"
    Then 系统返回扣减成功状态 (200 OK)
    And 药房主库中 "A司匹林" 的可用库存被标记为 98 盒
    And 产生一条包含事务号的审计流水 (Type: DISPENSE)

  Scenario: [Error] 库存不足导致的并发扣减阻断 (400)
    Given 药房主库中药品 "感冒灵" 当前总可用库存为 1 盒
    When 医生 "Doc_001" 要求开出 2 盒 "感冒灵"
    Then 系统必须阻止该写入事务并抛出 "ERR_INSUFFICIENT_STOCK"
    And 界面弹出红色提示 "库存不足，当前余量 1，请修正处方"
    And 药房数据库内 "感冒灵" 库存保持 1 盒不变，严禁超卖
```

**原因**: 可直接映射给测试特种兵当做用例，消除了任何开发者的脑补空间。

## 终级核验准则 Checklist (Exit Gates)

- [ ] PRD 是否完全抛弃了前端设计实现细节，只定义业务流转？
- [ ] BDD 是否至少包含了一个 [Success] 黑盒测例与一个 [Error] 阻断测例？
- [ ] 所有名词实体（如“药房库”、“处方权”）在全文中是否统一使用，没有产生分歧？
